# Intake Team — Premium UI Design Specification

**Version:** 2.0  
**Date:** 2026-06-09  
**Status:** Approved  
**Product:** Intake Team by Etherion Labs

---

## Executive Summary

Intake Team is a virtual receptionist for auto repair workshops (initial vertical: upholstery/tapicería). The UI must feel premium, modern, and invisible as software—users should feel they hired a digital receptionist, not that they bought a CRM.

**Key principle:** The interface should make it VISUALLY EVIDENT that an active digital receptionist is working for the user RIGHT NOW. Every interaction, every piece of data, every visual element should communicate "your receptionist just did something."

This spec defines the visual design, layout, components, and user experience for three core screens: **Inbox**, **Incoming**, and **Settings**.

---

## Core Philosophy

**"The user never feels they're operating AI. They feel they have a digital receptionist working for them."**

- ✅ IA is invisible
- ✅ Interface is operation-focused (conversations + jobs), not feature-focused
- ✅ Experience is clean, modern, premium SaaS (Linear, Notion, Stripe, Intercom, Arc)
- ❌ No traditional CRM appearance
- ❌ No admin panel aesthetic
- ❌ No overcomplicated navigation

## The "Wow Moment" — Active Receptionist Indicator

**Goal:** When the user opens Intake Team, they should feel that a digital receptionist is ACTIVELY WORKING for them RIGHT NOW.

### Visual Elements

#### 1. Pulsing "Receptionist Working" Indicator
Top-right of artifact panel, pulsing when agent is active:

```
🔷 ◐ Receptionist Processing...
```

- Icon: brand-secondary (cyan)
- Animation: Pulse (opacity 0.3 → 1) over 2 seconds
- Text: 11px, text-tertiary
- Updates in real-time

#### 2. Operational Summary Narrative
Status bar shows real-time actions:

```
"Just asked 3 clients for photos. María sent the budget. 2 jobs ready for review."
                                                    — Your Digital Receptionist
```

- Updates every time the agent takes an action
- Fades after 10 seconds (user can keep reading or dismiss)
- Communicates active work in natural language

#### 3. Timeline of Recent Actions
In artifact panel, shows what the receptionist just did:

```
ACCIONES DEL RECEPCIONISTA
🕐 2:15 PM (just now)  Pidió fotos
    ↳ "¿Podrías enviar imágenes del asiento?"
    ◐ Esperando respuesta...
```

- Most recent action at top
- Icon shows state (⏳ waiting, ✓ done, ◐ processing)
- Indented detail lines
- Timestamp shows "just now" vs relative time

#### 4. Photo Thumbnails Gallery
When client sends photos, they appear instantly:

```
FOTOS RECIBIDAS
4 imágenes
[📷][📷][📷][📷]
```

- Visual proof that client is engaged
- Grid of thumbnails, hover to expand
- Communicates: "your receptionist is collecting evidence"

#### 5. Urgency Badges in Kanban
Cards show how long they've been pending:

```
[Card] 🔴 URGENTE · 26h        ← Urgent attention needed
[Card] 🟠 PENDIENTE · 8h       ← Coming up
[Card] ⏳ 2h                     ← Still fresh
```

- Red for truly urgent (>24h)
- Amber for attention-worthy (>6h)
- Gray for normal (~2h)
- Creates visual urgency in pipeline

#### 6. Real-Time Progress Bar
Artifact shows intake completion live:

```
INTAKE COMPLETADO
6 / 8 campos · 75%
[████████░░] Growing...
```

- Bar fills as receptionist gathers info
- Fraction updates in real-time
- Shows receptionist is adding data

### Animation Sequence (On First Load)

1. **0-1s:** Artifact panel slides in from right (subtle, 300ms ease)
2. **1-2s:** "Receptionist Working..." indicator pulses (draws attention)
3. **2-3s:** Operational summary fades in (narrative appears)
4. **3-5s:** Timeline populates with recent actions (cards slide in)
5. **5-10s:** Photo gallery loads (if photos exist)

**Overall effect:** User feels like something important just happened, and the receptionist is actively engaged.

### Feedback Loops

**When receptionist takes action:**
1. Operational summary updates (bottom of status bar)
2. Timeline adds new entry (artifact panel)
3. Progress bar fills
4. Kanban cards may move
5. Artifact "Receptionist Working" pulsing activates for 3 seconds

**When something is urgent:**
1. Card gets red badge + left border
2. Kanban column may highlight
3. Optional: subtle notification pulse (if too urgent)

### Visual Language

**The receptionist should feel:**
- ✅ Busy (things changing, updates flowing)
- ✅ Competent (clear actions, structured information)
- ✅ Responsive (immediate visual feedback)
- ✅ Human (narrative language, timestamps, context)
- ✅ Trustworthy (premium aesthetic, careful detail)

NOT:
- ❌ Overwhelming (animations are subtle, not flashy)
- ❌ Technical (no prompts, models, workflows shown)
- ❌ Cold (no generic status messages)
- ❌ Slow (all updates feel instant)
