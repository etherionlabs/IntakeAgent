# Intake Team — Premium UI Design Specification

**Version:** 2.0  
**Date:** 2026-06-09  
**Status:** Approved  
**Product:** Intake Team by Etherion Labs

---

## Executive Summary

Intake Team is a virtual receptionist for auto repair workshops (initial vertical: upholstery/tapicería). The UI must feel premium, modern, and invisible as software—users should feel they hired a digital receptionist, not that they bought a CRM.

This spec defines the visual design, layout, components, and user experience for three core screens: **Inbox**, **Incoming**, and **Settings**.

---

## Core Philosophy

**"The user never feels they're operating AI. They feel they have a digital receptionist working for them."**

- ✅ IA is invisible
- ✅ Interface is operation-focused (conversations + jobs), not feature-focused
- ✅ Experience is clean, modern, premium SaaS (Linear, Notion, Stripe, Intercom, Arc)
- ❌ No traditional CRM appearance
- ❌ No admin panel aesthetic
- ❌ No overcomplicated navigation

## Design System

### Color Palette

```
Primary Colors:
--bg-primary: #ffffff              (Main background)
--bg-secondary: #f8f9fa            (Secondary surfaces, hover states)
--bg-tertiary: #f0f2f5             (Tertiary, grouping)
--text-primary: #0a0e27            (Headings, primary text)
--text-secondary: #6b7280          (Body text, secondary info)
--text-tertiary: #9ca3af           (Labels, hints, metadata)
--border-color: #e5e7eb            (Dividers, borders)
--border-light: #f3f4f6            (Light borders, subtle)

Semantic:
--accent: #2563eb                  (Primary action, focus)
--accent-danger: #dc2626           (Urgent, requires attention)
--accent-success: #10b981          (Completed, approved)
--accent-warning: #f59e0b          (Pending, in progress)
```

### Typography

```
Font Family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Roboto', sans-serif

Sizes & Weights:
- H1 (Page Title):        18px / 600 / Line 1.5
- H2 (Section Title):     16px / 600 / Line 1.5
- Body (Main content):    13px / 400 / Line 1.5
- Small (Metadata):       12px / 500 / Line 1.4
- Tiny (Labels):          11px / 600 / Line 1.2 / UPPERCASE
- Input/Button:           13px / 600 / Letter-spacing 0.5px
```

### Spacing Scale

```
xs: 4px    sm: 8px   md: 12px   lg: 16px   xl: 20px   2xl: 24px   3xl: 32px   4xl: 40px
```

### Shadows

```
--shadow-sm:   0 1px 2px 0 rgba(0, 0, 0, 0.05)
--shadow-md:   0 4px 6px -1px rgba(0, 0, 0, 0.1), 0 2px 4px -1px rgba(0, 0, 0, 0.06)
--shadow-lg:   0 10px 15px -3px rgba(0, 0, 0, 0.1), 0 4px 6px -2px rgba(0, 0, 0, 0.05)

Usage:
- Cards: shadow-sm on rest, shadow-md on hover
- Dropdowns/modals: shadow-lg
- No shadow by default; add on interaction
```

### Border Radius

```
Small elements (checkboxes, badges):     3-4px
Cards, inputs, buttons:                  6px
Containers, sections:                    8px
```

### Transitions

```
Default:  all 0.2s ease
Fast:     all 0.15s ease  (hover states)
Slow:     all 0.3s ease   (progress bars, important changes)
```

### Desktop (>1200px)
- Sidebar: 64px, fixed left
- 3-column layout (Inbox) or Kanban full-width (Incoming)
- All panels visible

### Tablet (768px - 1200px)
- Sidebar: 64px, still fixed left
- 2-column (List + Chat, no Artifact on side)
- Artifact accessible via modal or collapse/expand

### Mobile (<768px)
- Sidebar: Horizontal, 44px height, flex-row
- Single column main content
- Stacked panels (List → Detail)
- Bottom navigation optional
- Artifact as drawer/modal

## Interaction Patterns

### Hover States
- Cards: Border → accent, Shadow → md
- Buttons: Slight darkening or opacity change
- Nav items: Background → tertiary (inactive), Active stays accent

### Transitions
- All transitions: 0.2s (default)
- Progress bars: 0.3s (important changes)
- Rapid interactions (message send, toggle): 0.15s

### Feedback
- Success (mark ready, approve): ✅ Color green, optional toast
- Error: ❌ Color red, optional toast
- Loading: Subtle spinner or opacity change

---

## Accessibility

- ✅ Semantic HTML (buttons, labels, inputs)
- ✅ Color contrast: WCAG AA minimum (text-primary on bg-primary, etc.)
- ✅ Focus states: Keyboard navigation supported
- ✅ Labels: All inputs have associated labels
- ✅ Icons: Paired with text or title attributes
- ✅ Scrollbars: Custom but usable (thin, contrast 3:1 minimum)
